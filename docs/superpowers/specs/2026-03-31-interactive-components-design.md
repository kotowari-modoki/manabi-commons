# インタラクティブコンポーネント 設計書

**日付:** 2026-03-31
**ブランチ:** design/interactive-components
**ステータス:** 承認済み

---

## 背景・目的

まなびコモンズを「読むだけ」から「体験できる」サイトにするため、ページ内に直接埋め込めるインタラクティブコンポーネントを追加する。

- **対象ユーザー:** 小学生・中学生（一人で使う）
- **優先機能:** 練習問題・クイズ、図解アニメーション
- **後回し:** 進捗・達成バッジ（localStorage で後から足しやすい）

---

## アーキテクチャ

### 技術方針

- **MDX + Astro Island** を採用（Starlight は MDX をネイティブサポート）
- `client:visible` で遅延ロード（ページ初期表示速度を維持）
- スタイルは Starlight の CSS 変数に乗せて既存テーマと統一
- GitHub Pages の静的ホスティングのまま動作する（サーバー不要）

### ファイル構成

```
src/
├── components/
│   ├── Quiz.astro           # 練習問題コンポーネント
│   └── AnimatedStep.astro   # ステップ送りコンポーネント
└── content/docs/
    └── **/*.mdx             # 既存 .md を必要に応じて .mdx に変換
```

## コンポーネント仕様

### Quiz.astro

#### Props

| prop | 型 | 必須 | 説明 |
|------|----|------|------|
| `question` | `string` | ✓ | 問題文 |
| `choices` | `string[]` | ✓ | 選択肢（2〜4個） |
| `answer` | `number` | ✓ | 正解のインデックス（0始まり） |
| `explanation` | `string` | — | 正解後に表示する補足（任意） |

#### 動作仕様

1. 選択肢をクリック → 即時に正解・不正解を表示
2. 正解: 緑ハイライト＋⭐ アニメーション
3. 不正解: 赤ハイライト＋「もう一度」ボタンで再挑戦可能
4. `explanation` が指定されていれば正解後に補足テキストを表示
5. 正解済み状態を `localStorage` に保存（進捗機能の土台）

#### デザイン方針

ゲーム風・カラフル型。グラデーション背景、丸みのあるボタン、選択時のアニメーションで子どもが楽しめる演出。

#### 使用例

```mdx
import Quiz from '@/components/Quiz.astro'

<Quiz
  question="3 × 7 はいくつ？"
  choices={["18", "21", "24", "28"]}
  answer={1}
  explanation="3の段は3・6・9・12…と3ずつ増えます"
/>
```

### AnimatedStep.astro

#### Props

| prop | 型 | 必須 | 説明 |
|------|----|------|------|
| `title` | `string` | — | ステップ群のタイトル（任意） |
| `steps` | `{label: string, content: string}[]` | ✓ | 各ステップのデータ |

#### 動作仕様

1. 最初はステップ1のみ表示
2. 「次へ →」ボタンで次ステップを表示（前ステップは薄く残る）
3. 「← 前へ」ボタンで任意のステップに戻れる
4. 最後のステップで「まとめを見る」ボタン → 全ステップを一覧表示

#### デザイン方針

丸付き番号＋カード形式。番号はグラデーションカラー。「次へ」ボタンはゲーム風スタイルで統一。

#### 使用例

```mdx
import AnimatedStep from '@/components/AnimatedStep.astro'

<AnimatedStep
  title="繰り上がりのある足し算"
  steps={[
    { label: "① 一の位を計算", content: "8 + 7 = 15。5を書いて1を繰り上げます。" },
    { label: "② 十の位を計算", content: "3 + 4 = 7。さらに繰り上げた1を足して8。" },
    { label: "③ 答え", content: "38 + 47 = **85**" }
  ]}
/>
```

## コンテンツ編集フロー

1. 既存の `.md` ファイルを `.mdx` にリネーム
2. ファイル先頭に `import` 文を追加
3. 本文中の適切な場所にコンポーネントを埋め込む

既存の Markdown 記法（見出し、箇条書きなど）はそのまま使える。
